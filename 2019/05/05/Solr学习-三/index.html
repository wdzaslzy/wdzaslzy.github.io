Solr学习(三)

接上文：Solr学习（一）

上文主要总结了一下Solr的用途以及简单的使用方式。这儿将详细的总结一下它的一些管理操作。

Solr控制脚本

solr的控制脚本，在bin目录下，就是solr。上文了解到了solr的启动和停止方式。现在再介绍一些更详细一点的操作。

start参数

使用.\solr start -help命令，可以看到solr的start有很多参数。捡一些比较重要的参数说明一下。
solr start -a “string”
-a 参数后面可以指定一些JVM的参数。最长见的就是指定最大内存最小内存等。主要目的是用来优化solr运行。例如：

.\solr start -a “-Xms1024m”

solr start -c
-c 表示以SolrCloud的模式启动solr。在集群环境，需要连接到zookeeper。单纯使用-c，zookeeper会启用solr自带的zk。一般在集群环境，zookeeper是已经搭建好的，所以此处不需要solr带的zk，在-c后面增加-z即可。我的需求，就是使用SolrCloud模式。例如：

.\solr start -c -z zkServer1:2181,zkServer2:2181

solr start -
